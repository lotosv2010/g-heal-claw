# g-heal-claw 需求规格说明书（PRD）

> 版本: 2.0.0 | 日期: 2026-04-27 | 对应技术规格：`docs/SPEC.md`
>
> **文档层级：PRD（什么）→ SPEC（契约）→ ARCHITECTURE（拓扑）→ DESIGN（为什么）**

## 1. 项目背景

前端监控平台（Real User Monitoring，RUM）是一站式前端可观测解决方案，专注于 Web、H5、小程序等大前端领域，旨在通过自动化的数据采集与智能分析，实时掌握应用的真实用户体验，快速发现并定位问题。

## 2. 核心功能需求

### 2.1 性能监控

**目标**：全方位监控页面加载性能，量化用户体验。

**功能要点**：

| 功能项 | 说明 |
|--------|------|
| **Core Web Vitals 监控** | LCP（≤2.5s 良好）、FCP（≤1.8s 良好）、CLS（≤0.1 良好）、INP/FID（≤100ms 良好）、TTFB（≤200ms 良好） |
| **页面加载阶段监控** | DNS 查询、TCP 连接、SSL 建连、请求响应、内容传输、DOM 解析、资源加载等各阶段耗时 |
| **首屏时间监控** | 监听页面打开后指定时间窗口内的 DOM 变化，自动计算首屏渲染完成时间 |
| **卡顿监控** | 基于 PerformanceObserver API 检测长任务（2-5s）和无响应事件（>5s），反映页面流畅度 |
| **页面加载瀑布图** | 可视化展示页面加载各阶段的时序关系，帮助定位性能瓶颈 |

### 2.2 异常监控

**目标**：自动捕获前端运行时各类错误与异常，提供完整的错误上下文，实现快速定位与修复。

**功能要点**：

| 功能项 | 说明 |
|--------|------|
| **JS 运行时错误捕获** | 全局捕获未处理的 Error、TypeError、ReferenceError 等 |
| **Promise 未处理拒绝** | 捕获 unhandledrejection 事件 |
| **静态资源加载错误** | 监控脚本、样式、图片等资源加载失败 |
| **AJAX/Fetch 请求异常** | 监控请求错误（HTTP 状态码异常、超时、网络错误等） |
| **错误自动分类** | 按类型自动聚合：JS 错误、资源错误、API 错误、业务错误 |
| **错误上下文信息** | 错误堆栈、发生页面 URL、用户操作轨迹、设备与浏览器信息 |
| **Source Map 解析** | 支持上传 Source Map，将压缩代码堆栈还原为可读的源码位置 |
| **白屏检测** | 检测页面关键节点是否发生白屏，自动上报 |

### 2.3 API 监控

**目标**：追踪所有 XHR 和 Fetch 请求，监控接口的调用量、成功率、响应时间及异常。

**功能要点**：

| 功能项 | 说明 |
|--------|------|
| **请求自动拦截** | 通过劫持 XMLHttpRequest 和 fetch，无需业务代码改造 |
| **核心指标采集** | 请求量（QPS/TPM）、成功率、错误率、响应时间（平均值/P50/P90/P99）、超时率 |
| **异常请求追踪** | 记录异常请求的 URL、请求参数、响应状态码、响应内容（部分）、发生时间 |
| **慢请求识别** | 支持自定义阈值（如 >2s），自动标记慢请求用于优化分析 |
| **多维度分析** | 支持按接口地址、域名、状态码、地域、设备等维度聚合分析 |
| **链路上报** | 支持接入后端的 TraceID，实现前后端全链路追踪 |

### 2.4 页面访问监控

**目标**：掌握页面的真实访问数据，了解用户来源与行为路径。

**功能要点**：

| 功能项 | 说明 |
|--------|------|
| **PV/UV 统计** | 页面浏览量（PV）和独立访客数（UV），支持按页面路径、时间段、地域等维度分析 |
| **用户会话追踪** | 以用户会话为维度，记录用户访问路径与操作序列，还原真实使用场景 |
| **访问来源分析** | 页面来源（referrer）、搜索引擎来源、渠道来源等 |
| **终端环境采集** | 浏览器、操作系统、设备类型、屏幕分辨率、网络类型等 |
| **地域分布** | 基于 IP 的地理位置信息统计，支持全国/省份/城市维度的访问分布 |

### 2.5 静态资源监控

**目标**：监控 JS、CSS、图片、字体等静态资源的加载性能与可用性。

**功能要点**：

| 功能项 | 说明 |
|--------|------|
| **资源加载测速** | 通过 PerformanceResourceTiming API 采集各类资源的加载耗时 |
| **资源加载成功率** | 监控静态资源的加载失败次数与失败率 |
| **CDN 测速** | 针对通过 CDN 分发的资源进行专项测速，分析 CDN 节点性能 |
| **资源大小监控** | 统计各类资源（JS、CSS、图片）的总大小与请求数量 |
| **按资源类型拆分** | 支持按 script、style、image、font、media 等类型分类统计 |

### 2.6 自定义上报

**目标**：为业务方提供灵活的自定义数据上报能力，补充 SDK 无法自动采集的业务数据。

**功能要点**：

| 功能项 | 说明 |
|--------|------|
| **自定义事件上报** | 提供 `track(eventName, data)` 通用 API，上报任意业务事件 |
| **自定义测速上报** | 提供 `time(name, duration)` API，上报业务自定义耗时指标 |
| **自定义日志上报** | 支持上报分级日志（info / warn / error），可按级别过滤 |
| **Key-Value 键值对** | 支持附加业务自定义的键值对数据，满足灵活的数据统计需求 |
| **全局属性设置** | 支持设置全局上报属性（如用户 ID、版本号等），自动附加到所有上报事件中 |

### 2.7 埋点

**目标**：满足业务精确的用户行为分析需求，支持代码埋点、可视化埋点、全埋点三种方式。

**功能要点**：

| 功能项 | 说明 |
|--------|------|
| **代码埋点** | 提供简洁的 SDK API，在关键用户交互处手动调用上报（如按钮点击、页面跳转） |
| **无埋点/全埋点** | 通过监听全局事件（click、scroll 等），对含 `data-track` 属性的元素自动上报 |
| **可视化埋点** | 提供可视化圈选界面，运营/产品人员无需代码即可配置埋点事件 |
| **曝光埋点** | 基于 IntersectionObserver 监听元素进入/离开可视区，自动上报曝光事件 |
| **页面停留时长** | 自动记录用户在页面的停留时间 |
| **埋点数据规范** | 统一的事件命名规范、数据格式与字段约定，确保数据一致性 |

## 3. 非功能性需求

| 类别 | 需求描述 |
|------|----------|
| **上报策略** | 支持 sendBeacon（页面离开时）、fetch + 批量队列（SPA 场景）、Image 请求（兜底方案）三种上报方式 |
| **数据可靠性** | 本地缓存队列 + 失败重试（≥3 次），确保异常场景下数据不丢失 |
| **性能开销** | SDK 初始化需尽早执行（建议在 `<head>` 标签内），对页面性能影响控制在 2% 以内 |
| **采样率控制** | 支持配置采样率（如 0.1-1.0），在数据量与成本之间平衡，生产环境建议 0.1-0.3 |
| **敏感数据过滤** | 支持 `beforeSend` 钩子过滤请求中的 Token、密码等敏感信息，避免数据泄漏 |
| **多平台支持** | Web、H5、小程序（微信/支付宝）、移动端 App 等多平台统一接入与数据看板 |
| **多环境区分** | 支持按 `environment` 字段区分开发、测试、预发、生产环境，数据相互隔离 |
| **告警能力** | 支持配置告警规则：错误率突增（>5%）、关键页面性能劣化、API 成功率下降等，可通过钉钉/企微/邮件/短信通知 |
| **API 开放** | 提供开放 API 接口，便于与已有运维系统、数据平台集成 |

## 4. 监控看板（Dashboard）需求

- **核心指标卡**：页面 PV/UV、Apdex 评分、页面加载平均耗时、JS 错误率、API 成功率（概览面板）
- **趋势分析图**：各核心指标的时间序列趋势图，支持自定义时间段
- **多维分析视图**：按页面路径、地域、设备、浏览器、网络类型等维度下钻分析
- **Top 排行**：访问量 Top 页面、慢加载 Top 页面、错误数 Top 页面、慢请求 Top API
- **实时大盘**：实时数据刷新，适用于大促活动等关键场景的实时观测
- **单次会话详情**：支持查看单个用户会话的完整访问路径、操作序列与关联异常
