# 用户留存

路径：埋点分析 → **用户留存** `/tracking/retention`

> 状态：建设中（Phase 6 交付）

## 能力规划

基于 `track_events_raw` 的同期群留存分析：

- **次日 / 7 日 / 30 日留存**：标准口径
- **自定义锚点事件**：可指定"首次触发 X 事件"作为进入同期群的判定
- **留存热力矩阵**：按周 / 月滚动展示各同期群的留存衰减
- **分维度对比**：按渠道 / 设备 / release 拆分留存曲线

## 在此之前

留存依赖稳定的 `userId`。若当前业务仅有匿名 `sessionId`，留存口径会退化为 session 留存（跨天会丢失同一用户的关联），建议提前通过 `GHealClaw.setUser({ id })` 绑定用户。
