# 告警规则

路径：系统设置 → **告警规则** `/settings/alerts`

> 状态：建设中（Phase 4 交付）

## 能力规划

按指标 / 阈值 / 时间窗口创建告警，触发后分发到已配置的 [通知渠道](/guide/settings/channels)。

### 触发条件

告警触发条件的完整口径见 [接口说明 · 异常分析指标 · 告警触发条件](/reference/error-metrics#告警触发条件)。

### 最佳实践

| 场景 | 建议配置 |
|---|---|
| 新增异常即告警 | 首次出现 → 邮件 + 飞书 |
| 突增告警 | 1 分钟内异常数 > 过去 1 小时均值的 5 倍 |
| Core Web Vitals 恶化 | LCP P75 连续 10 分钟 > 4s |
| 5xx 飙升 | 错误率 > 5% 持续 2 分钟 |

### 降噪策略

- **Cooldown**：同一规则触发后冷却 N 分钟，避免刷屏
- **聚合合并**：同一 Issue 的多次触发只发一条摘要
- **静默窗口**：发布期 / 维护期可手动静默
