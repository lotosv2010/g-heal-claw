# 产品介绍

g-heal-claw 是一个面向前端应用的**监控与自愈平台**。它帮助你：

- **看见问题**：自动采集异常、性能、API、资源、访问与埋点数据
- **定位问题**：按指纹聚合为 Issue，叠加 Sourcemap 还原原始堆栈
- **修复问题**：AI 助手自动诊断根因，为关键问题生成修复 PR

## 它能为你做什么

| 角色 | 场景 |
|---|---|
| **前端工程师** | 上线后第一时间收到异常告警；打开 Issue 即可看到源码行号和面包屑回放 |
| **产品经理** | 在「访问分析」「埋点分析」查看关键路径转化与留存 |
| **技术负责人** | 在「Dashboard」查看整体健康度趋势；用 AI 自愈 PR 降低长尾问题的人力成本 |
| **SRE / 运维** | 告警规则与通知渠道配置化，线上回滚前先看「实时监控」 |

## 覆盖的监控能力

| 模块 | 内容 |
|---|---|
| 异常监控 | JS 异常 / Promise 拒绝 / 资源加载失败 / HTTP 异常 |
| 性能监控 | Core Web Vitals（LCP / INP / CLS）+ Navigation / Long Task |
| API 监控 | 请求耗时分布、慢接口、错误率、Top N |
| 资源监控 | 资源失败率、大小分布、类型聚合 |
| 访问分析 | UV / PV / 停留时长 / 跳出率 |
| 埋点分析 | 代码埋点 / 曝光 / 漏斗 / 留存 / 自定义上报 |

## 支持的端

- Web / H5
- 小程序（微信 / 支付宝 / 字节 · Phase 6）
- Native（通过 Hybrid WebView）
