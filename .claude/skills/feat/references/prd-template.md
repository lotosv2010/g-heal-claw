## 使用指南

### 何时使用 PRD

- 新增面向用户的功能（如 Dashboard 页面、SDK 插件、通知渠道）
- 涉及 UI 交互变更
- 需要跨团队/跨角色对齐需求理解

### 与 ARD 的关系

- 如果需求同时涉及产品功能和接口设计 → 先写 PRD，再写 ARD
- 如果需求纯后端/纯接口 → 直接用 ARD，跳过 PRD
- 本项目中，PRD 内容通常已覆盖在 `docs/PRD.md` 中，Phase 1 读取即可，无需从零编写
